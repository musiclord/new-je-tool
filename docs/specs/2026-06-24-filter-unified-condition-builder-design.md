# Step 4 進階篩選 統一條件建構器（單一介面、分組融入、新手友善）

> **狀態：設計提案，使用者已核可方向（含整句回顯＋4 拍提示）。** 純前端（`filter-step.js` / `ui-core.js` / `app.css`），不動後端、不動 wire 契約（仍兩層 `groups` AST）。承 r6，回應使用者：要**一套以簡易檢視為基礎的乾淨介面**、把「條件群組」融入、**移除「進階：分組」模式切換**，並讓非專業者一眼懂流程。

## 背景

r6 把扁平檢視做得乾淨（well＋整句 radio），但 r6 仍保留「簡易⇄進階」兩套，且「進階：分組」會新增空群組、回不去、還報錯。使用者要求：**只有一套介面**，分組是常態能力（不是模式），且整個 Step 4 對非專業者自我解釋。

三路網路研究收斂（皆有來源）：
- **HubSpot 兩層扁平群組**為最佳範本：一組內 AND、組間 OR；「＋加入條件」「＋新增群組」並列常駐按鈕、**非模式**；一組時就是乾淨一塊。Airtable/Notion 同為「加組是普通按鈕、就地升級、封頂 2–3 層」。
- **一組只用一種組合器**（要混→再開一組）；**封頂兩層**；學術（VQuery）警告花俏巢狀 builder 對新手更慢更易錯。
- **全程白話**（符合 全部/任一），不用 AND/OR/群組/巢狀術語；組間連接器白話＋上色（Google Ads 具名區塊）。
- **空狀態要教學**（NN/g 三職責）、**整句回顯**（read-back）防呆、**漸進揭露**。

## 設計

### 1. 模型：情境 ＝ 一個或多個「條件組（set）」，每組就是一塊 well

- **每個 set ＝ 一塊淡底 well**：頂部「符合 全部／任一」（整句 radio，**該組 ≥2 條件才顯示**）＋條件清單。
- **一組（最常見）＝ 跟 r6 一樣乾淨**：看不到「組」字、無連接器、無 remove-set。
- **「＋ 另一組條件」**：常駐普通按鈕（取代「進階：分組」，**無模式切換**）。新增一個 set。
- **≥2 組**：set 之間出現**白話連接器**「符合任一組即可／需符合所有組」（上色、漸進，只在 ≥2 組時）。每組有「移除這組」。
- **一組一種組合器**；要混 AND/OR → 再開一組。**封頂兩層**（一堆 set、每 set 一堆條件，不再更深）。

### 2. AST 映射（不改契約；仍 `draft.groups`）

- 「可編輯群組」（非預設）＝ 每個對應一塊 well。「預設群組」（`__kctPresetGroup`，如 I）→ 以**原子單行**呈現，**併入第一塊 well**（所以 G+I+J 仍是一塊乾淨 well，不會變兩塊）。
- 若無可編輯群組（只選了 I）→ 一塊 well 只含原子行、無組合器（單一原子條件）。
- **每塊 well 的 radio** 設「該可編輯群組各 rule 的 join」；**第一塊 well 的 radio 另把所有預設群組的 `join` 同步成同值**（讓原子行與其他條件以同一組合器結合）。
- **set 之間連接器**設「該（後一個）可編輯群組的 `join`」。
- **預設組合器**：組內預設 **全部(AND)**；**新增的 set 之間預設 OR（符合任一組即可）**（HubSpot 慣例：組間 OR＝命中任一替代組）；預設群組 I 的連接器跟隨第一塊 well 的組合器。

### 3. 防呆（解決 r6 的困住/報錯）

- **空群組送出時自動略過**：`toWireScenario` 過濾 `rules.length > 0` 的群組（建到一半的空 set 不再讓預覽報「群組 N 沒有規則」）。若全空→既有 `scenarioGate` 的 noCondition 擋下。
- **回得去**：移除 set 到只剩一個可編輯群組，畫面自動回到乾淨單 well（不需「回到簡易」鈕、無模式）。

### 4. 新手層（已核可）

- **教學空狀態**：無條件時，彙整區顯示「還沒有任何條件。↑ 到上面挑一張『KCT條件』卡，或展開『自訂篩選條件』；挑到的條件會落到這裡讓你設定數值。」（NN/g：狀態＋學習線索＋指向 palette）。
- **整句回顯（read-back）**：≥1 條件時，於條件下方常駐一句白話：「這個情境會找出：符合『全部』下列條件的分錄 — 金額尾數 000000；摘要空白。」多組時：「…符合『任一組』的分錄 — 第1組（… 全部）；第2組（…）。」用既有 `ruleSummaryLabel`／`presetAtomLabel` 組字。
- **4 拍流程輕提示**：彙整區標題下一行小字「挑訊號 → 設定數值 → 組合 → 命名保存」。
- **漸進揭露**：每組組合器只在該組 ≥2 條件時顯示（沿用 r6）。

### 5. 不變

- 純前端；wire `{name, rationale, groups}`；命中數與改版前等價（只是呈現統一）。
- 非營業日(I) 仍原子行；KCT 多選命名（字母串＋說明）不變。
- minimalist-ui（淡底 well、hairline、整句 radio、白話、無術語/重框/emoji）；中文；ES5。

## 驗收

- **G+H**：一塊乾淨 well（「這些條件要怎麼搭配？」＋兩 radio）＋下方 read-back 一句；無「組」字、無連接器。
- **G+I+J**：仍是**一塊** well（非營業日為一行原子），不會變兩塊。
- **＋ 另一組條件**：出現第 2 塊 well＋白話連接器「符合任一組即可／需符合所有組」（預設任一組）；每塊 well 各自「符合 全部/任一」。
- **移除這組** 到剩一組 → 自動回乾淨單 well（無「進階/簡易」字樣、無模式鈕）。
- **空 set 不報錯**（送出略過）；全空仍被必填閘擋下。
- **read-back** 隨條件/組合器即時更新且讀得通；**教學空狀態** 指向上方 palette；**4 拍提示** 顯示；組合器只在 ≥2 條件出現。
- 切組合器後〔預覽這個情境〕命中數隨布林邏輯改變；`toWireScenario` 仍乾淨（無 UI-only 鍵）。
- `dotnet build` 綠；GUI 目視列入 `docs/windows-handoff.md`。
