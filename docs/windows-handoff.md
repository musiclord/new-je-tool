# Windows 手動驗證清單

這是一份滾動式交接清單，只列「必須在實際 app（WinForms／WebView2）裡親手點過才能確認」的項目。自動化測試驅動不了 GUI，所以這些項目只能靠人工目視。自動化測試（`dotnet test`）由 agent 自跑，目前全綠。

新增任務卡時請寫成自含的一張卡，四個欄位缺一不可：啟動指令、涵蓋內容、預期結果、失敗回報格式。失敗回報要寫清楚三件事——在哪一步、做了什麼、看到什麼；若 F12 console 有錯，一併附上。共通的啟動指令是 `dotnet run --project src/JET/JET`（Debug 組建）；多數項目可先用「套用測試案件」按鈕備妥資料。某張卡驗收通過後，把它移到下方「已完成」段。

## 待驗收

- **匯出底稿存檔對話框與端到端匯出（子專案 E1，匯出底稿 writer；Task 6）**
  - 啟動：`dotnet run --project src/JET/JET`（Debug）。先在「匯入資料」用「套用測試案件」備妥 GL/TB，在「欄位配對」完成 GL/TB 配對，在「資料驗證與測試」按「執行驗證」，並在「進階條件篩選」保存至少一個會命中的情境，好讓 step3／step4 矩陣有資料。
  - 涵蓋內容：到「匯出底稿」步驟，按〔選擇位置並匯出底稿〕。
    1. 應彈出原生 SaveFileDialog（`host.selectSavePath`），filter 為「Excel 活頁簿 (*.xlsx)」，預填檔名為 `{公司名}_{yyyymmddHHmmss}_WorkingPaper.xlsx`（公司名取自當前專案，時間戳是當下時間）。
    2. 選一個路徑存檔後，後端會串流寫出（`export.workpaperStream`），面板顯示完成統計：存檔路徑、檔案大小（位元組）、各工作表的資料列數表。
    3. 取消 SaveFileDialog 時，訊息面板應顯示「已取消匯出底稿。」，不寫檔、不報錯。
  - 預期結果：
    - 預填檔名的時間戳是按下當下產生的（不是固定值）；取消對話框能安全中止。
    - 寫出的 `.xlsx` 用 Excel 開啟時不應出現修復提示。內容應含封面（公司名等於當前專案的客戶名）、step1 家族、step2 可靠性、step3／step4／step4-1 高風險矩陣（與已保存情境對得起來）、step5、以及三張「自動化工具」參考表。
    - 完成統計裡各表的列數要與實際工作表的資料列數相符。step3 的命中傳票數要與「進階條件篩選」裡該情境單獨預覽的結果對得起來，藉此證明匯出前已 materialize。
  - 失敗回報格式：在哪一步（上列 1–3）、做了什麼、看到什麼；F12 console 有錯一併附上。常見問題例如：對話框沒彈出、預填名或 filter 不符、寫出的檔 Excel 打不開、公司名錯、矩陣與情境對不起來、完成統計列數不符、postMessage 例外。

- **高風險條件矩陣預覽（子專案 D2，多情境逐列 tag 矩陣；Task 5）前端**
  - 啟動：`dotnet run --project src/JET/JET`（Debug）。先在「匯入資料」用「套用測試案件」備妥 GL/TB，在「欄位配對」完成 GL/TB 配對，再到「進階條件篩選」保存**至少兩個會命中的情境**（可先用「套用測試案件」備一個，再手動加幾個 prescreen 或自訂條件情境，最多到 10 個）。
  - 涵蓋內容：「進階條件篩選」頁底多了一張「高風險條件矩陣」卡。按〔展開矩陣〕會惰性載入三段。
    1. **情境摘要**（step3）：一列對一個已存情境，也就是一個高風險條件欄，顯示「欄（C1..CN）／條件名稱／命中傳票數／命中行數」。欄序依後端回的 position 升冪。
    2. **傳票矩陣**（step4）：每列一張命中傳票，有固定欄（傳票號／總帳日／編製人員／傳票總額）加上動態的 C1..CN 欄。某傳票命中某條件時，該 C 欄打 ✓，否則留空。表格下方〔載入更多〕逐頁接續。
    3. **行層明細**（step4-1）：列出命中傳票的**所有行**，包含未命中任何條件的行。有固定欄（傳票號／項次／總帳日／科目／金額／摘要）加上動態的逐行 C tag（該行命中才打 ✓）。沒有單傳票過濾，靠傳票號欄對照。表格下方〔載入更多〕逐頁接續。
  - 預期結果：
    - C 欄數等於已存情境數；欄序與情境摘要的 position 升冪一致；C 欄標頭 hover 時用 tooltip 顯示情境名稱。情境到 10 個時欄位仍正確對位、不錯位。
    - 打 ✓ 或留空是由後端的 matchedPositions 決定的：某傳票或某行在哪些 C 欄打 ✓，要與情境摘要的命中數、以及該情境單獨〔預覽〕的命中傳票或行對得起來。
    - 傳票矩陣與行層各自的〔載入更多〕逐頁接續（≤200 列／頁）；接到底時鈕消失；載入中時鈕顯示「載入中…」並 disabled。大母體逐頁載入要流暢，寬矩陣在框內水平捲動、不整頁爆版。
    - 〔收合矩陣〕後再〔展開矩陣〕不應重抓（有快取）。沒保存任何情境時，矩陣卡顯示友善的空狀態、不出現展開鈕。
  - 失敗回報格式：在哪一段（摘要／傳票矩陣／行層）、做了什麼、看到什麼；F12 console 有錯一併附上。常見問題例如：C 欄數或欄序不符、✓ 對位錯、命中數對不起來、接續列不前進、接到底鈕沒消失、寬表爆版、postMessage 例外。
