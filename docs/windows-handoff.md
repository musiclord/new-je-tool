# Windows 手動驗證清單

滾動式交接:這裡只列「需要在實際 app(WinForms／WebView2)裡點過才能確認」的事——自動化測試驅動不了 GUI。自動化測試(`dotnet test`)由 agent 自跑、目前全綠。

新增任務卡時請自含:啟動指令、涵蓋內容、預期結果、失敗回報格式(在哪一步、做了什麼、看到什麼;F12 console 有錯一併附上)。啟動:`dotnet run --project src/JET/JET`(Debug;多數項目先用「套用測試案件」備妥資料)。驗收通過的卡片移到下方「已完成」段。

## 待驗收

- **WebView2 檔案對話框 reentrancy 崩潰修復(host 端)**
  - 啟動:`dotnet run --project src/JET/JET`(Debug)。
  - 背景:`host.selectFile` / `host.selectFiles` / `host.selectSavePath` 原本在 WebView2 的 `WebMessageReceived` 事件處理常式內**同步**開 `OpenFileDialog`／`SaveFileDialog`,觸發 WebView2 不支援的 reentrancy,瀏覽器程序以 `0x80000003` 崩潰、整個 app 閃退(無 .NET 例外)。已改為經 `Form1.ShowDialogDeferredAsync`(`BeginInvoke` 延到事件處理常式返回後再開)。權威見 WebView2 Threading model › Reentrancy、社群 WebView2Feedback #2946/#4648。
  - 涵蓋內容(反覆操作各檔案對話框,確認不再閃退):
    1. 「匯入資料」多來源精靈選**多個**檔案(`host.selectFiles`,.xlsx/.csv)。
    2. 行事曆假日/補班、科目配對、授權編製人員清單的**單檔**選擇(`host.selectFile`)。
    3. 「匯出底稿」的存檔對話框(`host.selectSavePath`)。
    4. 每個對話框各「開 → 選檔 / 取消」重複數次、含快速連點,確認都正常返回、無凍結、無閃退。
  - 預期結果:對話框正常彈出與返回(選檔與取消都正確),選到的路徑/檔名照常進入後續流程;**反覆操作不再閃退或凍結**;F12 console 無 postMessage 例外。
  - 失敗回報格式:在哪個對話框、操作幾次後、看到什麼(凍結/閃退/錯誤);若仍閃退,附 `%LOCALAPPDATA%\JET\logs\` 最新 `.ndjson` 末段,以及 Windows 事件檢視器 Application Error 的失敗模組與例外代碼。
